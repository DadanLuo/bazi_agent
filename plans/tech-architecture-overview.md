# 赛博司命 - 八字分析系统技术架构概述

## 1. 项目概览

**赛博司命**是一个基于 LangGraph 的智能八字分析 Agent 系统，采用现代化的全栈开发架构，结合传统命理学知识与现代 AI 技术。系统支持完整的八字排盘、五行分析、格局判断、喜用神分析、流年运势、大运分析等功能，并通过 RAG（检索增强生成）技术结合古籍知识库提供专业的命理分析报告。

## 2. 前端技术栈

### 2.1 核心技术
- **HTML5/CSS3**: 纯静态前端实现，无外部框架依赖
- **Vanilla JavaScript**: 原生 JavaScript 实现所有交互逻辑
- **响应式设计**: 支持桌面和移动设备

### 2.2 前端架构特点
- **单页面应用 (SPA)**: 所有功能在一个 HTML 文件中实现
- **组件化设计**: 通过 CSS 类名实现模块化布局
- **状态管理**: 通过 JavaScript 变量管理应用状态
- **API 集成**: 直接调用后端 RESTful API

### 2.3 主要功能模块
- **输入表单**: 出生信息收集（年月日时、性别、出生地）
- **实时加载指示器**: 显示分析进度和步骤
- **结果展示区**: 
  - 四柱八字排盘（年柱、月柱、日柱、时柱）
  - 五行分析图表（木火土金水柱状图）
  - 格局判断和喜用神分析
  - 流年运势和大运时间轴
  - AI 智能分析报告
- **RAG 知识检索面板**: 显示检索到的古籍片段
- **智能对话系统**: 支持多轮追问和上下文记忆

### 2.4 前端性能优化
- **纯静态资源**: 无构建步骤，直接部署
- **内联样式**: 减少 HTTP 请求
- **本地存储**: 支持报告下载为 Markdown 格式

## 3. 后端技术栈

### 3.1 核心框架
- **FastAPI**: 高性能 Python Web 框架，用于构建 RESTful API
- **LangGraph**: 基于 LangChain 的工作流编排框架，用于定义复杂的分析流程
- **Pydantic**: 数据验证和序列化，确保 API 输入输出的一致性

### 3.2 编程语言和运行环境
- **Python 3.10+**: 主要开发语言
- **Uvicorn**: ASGI 服务器，支持异步处理
- **Setuptools**: 项目打包和分发

### 3.3 核心依赖库
- **langgraph >= 0.2.0**: 工作流状态机和图执行引擎
- **langchain >= 0.3.0**: LLM 集成和工具链
- **dashscope >= 1.20.0**: 阿里云通义千问 API 客户端
- **chromadb >= 0.4.0**: 向量数据库，用于 RAG 知识存储
- **pydantic >= 2.12.0**: 数据模型和验证
- **redis >= 5.0.0**: 缓存和会话存储
- **cachetools >= 5.3.0**: 内存缓存工具
- **requests >= 2.31.0**: HTTP 客户端
- **python-dotenv >= 1.0.0**: 环境变量管理

## 4. AI 和机器学习集成

### 4.1 LLM 集成
- **通义千问 (Qwen-Plus)**: 主要使用的 AI 模型
- **DashScope API**: 阿里云提供的 LLM 服务接口
- **智能续写机制**: 当生成内容被截断时自动续写
- **安全检查**: 敏感内容过滤和心理危机检测

### 4.2 RAG (检索增强生成)
- **混合检索策略**: 结合向量检索和 BM25 关键词检索
- **向量数据库**: ChromaDB 存储古籍知识向量
- **BM25 检索**: 基于关键词的传统检索方法
- **重排序 (Reranking)**: 对检索结果进行相关性重排序
- **配置化检索模式**: 支持 vector_only、bm25_only、hybrid、hybrid_rerank 四种模式

### 4.3 知识库构建
- **古籍文本处理**: 将传统命理学典籍转换为结构化知识
- **向量化处理**: 使用嵌入模型将文本转换为向量
- **元数据管理**: 为每个知识片段添加分类和标签

## 5. 中间件和基础设施

### 5.1 缓存层
- **Redis**: 分布式缓存系统
  - 八字分析结果缓存（TTL: 2小时）
  - 会话数据缓存（TTL: 24小时）
  - 检索结果缓存（TTL: 5分钟）
- **内存缓存**: cachetools 提供的本地缓存
- **装饰器模式**: `@cached` 和 `@async_cached` 装饰器简化缓存使用

### 5.2 存储层
- **文件存储**: JSON 格式持久化会话数据
- **Redis 存储**: 高性能缓存和临时数据存储
- **ChromaDB**: 向量数据库，存储 RAG 知识库

### 5.3 安全中间件
- **CORS 中间件**: 处理跨域请求
- **内容安全检查**: 过滤敏感内容
- **心理危机检测**: 识别用户心理危机信号并提供援助

### 5.4 日志和监控
- **Python logging**: 标准日志记录
- **结构化日志**: 记录关键操作和错误信息
- **性能监控**: 记录 LLM 调用的 token 消耗

## 6. 系统架构设计

### 6.1 整体架构
```
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│   前端界面       │    │   FastAPI 后端   │    │   AI/存储服务    │
│  (HTML/CSS/JS)  │◄──►│  (RESTful API)  │◄──►│  (LLM/RAG/Cache)│
└─────────────────┘    └─────────────────┘    └─────────────────┘
        ▲                       ▲                       ▲
        │                       │                       │
        └───────────────────────┴───────────────────────┘
                    统一状态管理 (UnifiedStateManager)
```

### 6.2 工作流架构 (LangGraph)
系统采用 LangGraph 定义了完整的八字分析工作流：

1. **输入验证** → **八字排盘** → **五行分析** → **格局判断** → **喜用神分析**
2. **流年分析** → **大运分析** → **知识检索 (RAG)** → **LLM 报告生成** → **安全检查**

每个节点都是独立的函数，通过条件路由实现错误处理和流程控制。

### 6.3 会话管理
- **UnifiedStateManager**: 统一的状态管理器
- **双存储策略**: Redis (内存) + 文件存储 (持久化)
- **上下文压缩**: 长对话自动摘要压缩
- **意图识别**: 自动识别用户意图 (NEW_ANALYSIS, FOLLOW_UP, TOPIC_SWITCH 等)

### 6.4 缓存策略
- **跨会话缓存**: 相同出生信息的分析结果全局缓存
- **会话级缓存**: 单个会话的上下文缓存
- **检索结果缓存**: 相同查询的 RAG 结果缓存
- **智能失效**: 基于 TTL 的自动过期机制
